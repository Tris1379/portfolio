```typescript
const pageVariants = {
  initial: {
    opacity: 0,
    filter: "blur(8px)",
    y: 20,
  },
  animate: {
    opacity: 1,
    filter: "blur(0px)",
    y: 0,
    transition: { duration: 0.5, delay: 0.45, ease: [0.25, 0.46, 0.45, 0.94] },
    // delay 0.45s = leaves start dispersing, new content fades in
  },
  exit: {
    opacity: 0,
    filter: "blur(16px)",
    y: -10,
    transition: { duration: 0.35, ease: "easeInOut" },
    // faster exit (0.35s) = content gone by 0.35s, well before leaf peak at 0.4s
  },
};
```

Key timing:
- 0.0s: `isTransitioning=true` → storm particles spawn, `exit` animation starts on current page
- 0.35s: Current page fully faded out
- 0.4s: `setCurrentPage(newPage)` fires (from PageContext setTimeout) → new page mounts
- 0.45s: New page `animate` begins (delay in transition) → leaves start dispersing
- 0.95s: New page fully visible, `isTransitioning=false` → storm unmounts, particles disposed

**2c. The AnimatePresence `mode="wait"` handles the exit→enter sequencing automatically.** No manual coordination needed — the key={currentPage} change triggers exit on old, then enter on new.
